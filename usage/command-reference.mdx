## Synopsis

```bash
ansible-cmdb [options] <dir> > output.html
```

The `<dir>` argument specifies the directory containing Ansible fact files generated by `ansible -m setup --tree <dir> all`.

## Command-Line Options

<ParamField path="<dir>" type="string" required>
  Directory containing Ansible fact files or fact cache files. This is the output directory from `ansible -m setup --tree <dir>` command.
</ParamField>

<ParamField path="-t, --template" type="string" default="html_fancy">
  Template to use for output generation. Can be a template name or a path to a custom template file.
  
  See [Templates](/usage/templates) for available templates.
</ParamField>

<ParamField path="-i, --inventory" type="string">
  Path to Ansible inventory file, directory, or dynamic inventory script. Multiple inventories can be specified by separating them with commas (no spaces).
  
  This option enables ansible-cmdb to read:
  - Host groups
  - Host variables (including `dtap`, `comment`, `ext_id`)
  - Group variables from `group_vars/` and `host_vars/` directories
  
  See [Inventory Scanning](/usage/inventory-scanning) for details.
</ParamField>

<ParamField path="-f, --fact-cache" type="boolean" default={false}>
  Indicates that `<dir>` contains fact-cache files instead of regular setup output files.
  
  When enabled, this option applies to **all** fact directories specified. You cannot mix fact-cache and regular setup directories.
  
  See [Fact Caching](/usage/fact-caching) for details.
</ParamField>

<ParamField path="-p, --params" type="string">
  Parameters to pass to the template. Multiple parameters are separated by commas with **no spaces**.
  
  Format: `key1=value1,key2=value2`
  
  Example:
  ```bash
  ansible-cmdb -p local_js=1,collapsed=1 out/ > overview.html
  ```
  
  See [Templates](/usage/templates) for template-specific parameters.
</ParamField>

<ParamField path="-c, --columns" type="string">
  Comma-separated list of column IDs to display in the output. Only applicable to templates that support columns (e.g., `html_fancy`, `txt_table`).
  
  Example:
  ```bash
  ansible-cmdb -c name,os,ip,mem,cpus facts/
  ```
</ParamField>

<ParamField path="-C, --cust-cols" type="string">
  Path to a custom columns definition file. Allows you to define additional columns using jsonxs expressions or Mako templates.
  
  Only supported by `html_fancy` and `html_fancy_split` templates.
  
  Example:
  ```bash
  ansible-cmdb -C custom_columns.conf -i hosts out/ > cmdb.html
  ```
</ParamField>

<ParamField path="--exclude-cols" type="string">
  Comma-separated list of column IDs to exclude from the output. Works with templates that support columns.
  
  Example:
  ```bash
  ansible-cmdb --exclude-cols mem_usage,swap_usage facts/
  ```
</ParamField>

<ParamField path="-l, --limit" type="string">
  Limit hosts included in the output using Ansible-style patterns. Supports:
  - `all` - all hosts
  - Individual hosts: `host1.example.com`
  - Groups: `webservers`
  - Inversion: `!groupname`
  - Combinations: `all:!development:db1.example.com`
  
  Requires the `-i` option to be specified.
  
  See [Limiting Hosts](/usage/limiting-hosts) for details.
</ParamField>

<ParamField path="-d, --debug" type="boolean" default={false}>
  Enable debug output. Shows detailed logging information during execution.
</ParamField>

<ParamField path="-q, --quiet" type="boolean" default={false}>
  Suppress warning messages. Only error messages will be displayed.
</ParamField>

<ParamField path="--version" type="boolean">
  Display the program version number and exit.
</ParamField>

<ParamField path="-h, --help" type="boolean">
  Show help message with all options and exit.
</ParamField>

## Basic Usage Examples

<CodeGroup>
```bash Generate basic HTML overview
mkdir out
ansible -m setup --tree out/ all
ansible-cmdb out/ > overview.html
```

```bash Use specific template
ansible-cmdb -t txt_table out/ > overview.txt
```

```bash Include inventory information
ansible-cmdb -i ./hosts out/ > overview.html
```

```bash Limit to specific hosts
ansible-cmdb -i hosts -l 'webservers' out/ > webservers.html
```

```bash Custom columns only
ansible-cmdb -c name,os,ip,mem out/ > minimal.html
```
</CodeGroup>

## Advanced Examples

### Multiple fact directories

You can specify multiple directories to overlay facts:

```bash
ansible-cmdb out/ out_extend/ out_custom/ > overview.html
```

This allows you to:
- Override facts
- Add missing information
- Create manual host entries
- Add custom facts

### Combining options

```bash
ansible-cmdb \
  -t html_fancy_split \
  -i hosts \
  -p local_js=1,collapsed=1 \
  --exclude-cols mem_usage,swap_usage \
  -l 'production' \
  out/
```

## Notes

<Note>
  `ansible-cmdb` is a shell wrapper script that finds the correct Python binary and the `ansible-cmdb.py` script location. If it fails, you can manually specify:
  
  ```bash
  /opt/python3/bin/python /opt/ansible-cmdb/src/ansible-cmdb.py out/ > overview.html
  ```
</Note>

<Warning>
  When using `--fact-cache`, you cannot mix fact-cache directories with regular setup output directories. All specified directories must be of the same type.
</Warning>
